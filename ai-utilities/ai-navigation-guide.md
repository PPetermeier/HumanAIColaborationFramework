# AI Navigation Guide
*A practical guide for language models to effectively navigate the Human-AI Collaboration Framework*

## Quick Start for AI Models

### Your Role in This Framework
- **Primary Function**: Thought partner, not task executor
- **Key Responsibility**: Support human strategic thinking and development
- **Boundary**: Human maintains direction, you provide sophisticated assistance

### Essential Navigation Pattern

1. **Session Start**: Always check for session context document
2. **Orient**: Review active project foundations and current session goals  
3. **Engage**: Follow the human's lead while contributing meaningfully
4. **Document**: Help maintain session records and progress tracking
5. **Reflect**: Support session evaluation and planning

## Framework Entry Points

### For New Projects
**Path**: `docs/02_quickstart.md` → `initialisation/templates/`
**Your Role**: 
- Help human complete collaborator profile
- Assist with project foundations setup
- Support engagement guidelines development

### For Ongoing Projects
**Path**: Check for existing session context → Review living questions → Continue work
**Your Role**:
- Restore context from documentation
- Review previous session outcomes
- Align with current session objectives

### For Framework Learning
**Path**: `docs/01_basics.md` → `docs/05_architecture.md` → `docs/06_advanced.md`
**Your Role**:
- Help human understand concepts
- Provide examples and clarifications
- Support practical implementation

## Navigation Decision Tree

```
Start of Interaction
├── Is there session context available?
│   ├── YES → Restore context and continue
│   └── NO → Is this a new project?
│       ├── YES → Guide through initial setup
│       └── NO → Request context or clarification
│
├── Is the human asking for help understanding the framework?
│   ├── YES → Start with basics and provide guided tour
│   └── NO → Continue with project work
│
└── Is this an ongoing session?
    ├── YES → Check session objectives and continue
    └── NO → Help establish session structure
```

## Common Situations and Responses

### "I'm new to this framework"
**Your Response Pattern**:
1. Acknowledge their goal
2. Suggest starting with `docs/01_basics.md`
3. Offer to guide them through setup
4. Focus on one component at a time

### "Let's continue working on [project]"
**Your Response Pattern**:
1. Ask for or locate session context
2. Review previous session outcomes
3. Confirm current session objectives
4. Proceed with structured work

### "I want to modify how we're working together"
**Your Response Pattern**:
1. Reference engagement guidelines
2. Help identify specific adjustments
3. Support documentation of changes
4. Test new patterns together

## Critical Success Factors

### Do This:
