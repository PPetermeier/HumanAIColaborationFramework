# Quick Reference Cards
*Compact guides for each major framework component*

---

## 📋 Collaborator Profile Card

### Purpose
Document human working patterns, preferences, and expertise to inform collaboration approach.

### Key Sections
- **Technical Background**: Skills, experience, domain knowledge
- **Tool Proficiency**: Development tools, knowledge management, analysis tools
- **Working Style**: Problem-solving approach, learning patterns, cognitive preferences
- **Communication Preferences**: Interaction style, documentation, feedback patterns

### AI Usage Pattern
1. **Reference** at session start to understand human preferences
2. **Adapt** communication and interaction style accordingly
3. **Help maintain** and update as patterns evolve
4. **Use** to customize other framework components

### Critical Elements for AI
- Working style preferences (detailed vs. high-level, structured vs. flexible)
- Communication patterns (direct vs. exploratory, verbal vs. visual)
- Learning objectives and development areas
- Tool preferences and technical constraints

## 🎯 Project Foundations Card

### Purpose
Establish clear project parameters, objectives, and quality standards for focused collaboration.

### Key Sections
- **Purpose and Scope**: Core objectives, boundaries, success criteria
- **Context and Constraints**: Environment, limitations, requirements
- **Quality Framework**: Standards, evaluation criteria, review processes
- **Success Metrics**: Measurement approaches, milestones, completion criteria

### AI Usage Pattern
1. **Align** all suggestions and analysis with stated objectives
2. **Reference** constraints when evaluating options
3. **Support** quality standard maintenance throughout work
4. **Help track** progress against defined metrics

### Critical Elements for AI
- Primary objective (what success looks like)
- Key constraints (what must be considered)
- Quality standards (how to evaluate outputs)
- Scope boundaries (what's included and excluded)

## 📝 Session Template Card

### Purpose
Structure individual collaboration sessions for consistency, documentation, and progress tracking.

### Key Sections
- **Session Planning**: Objectives, approach, time allocation
- **Work Documentation**: Activities, decisions, outputs
- **Progress Tracking**: Accomplishments, open items, next steps
- **Reflection and Evolution**: Learning, adjustments, improvements

### AI Usage Pattern
1. **Follow** established session structure
2. **Maintain** documentation throughout session
3. **Support** progress tracking and planning
4. **Contribute** to reflection and improvement processes

### Critical Elements for AI
- Session objectives and success criteria
- Documentation requirements and formats
- Progress tracking mechanisms
- Handoff preparation for session continuity

## ❓ Living Questions Card

### Key Sections
- **Core Questions**: Fundamental project inquiries
- **Emerging Questions**: New areas of exploration
- **Resolved Insights**: Questions answered and learning gained
- **Question Evolution**: How inquiries develop over time

### AI Usage Pattern
1. **Reference** questions to guide analysis and suggestions
2. **Help identify** new questions emerging from work
3. **Support** exploration of key questions
4. **Contribute** to question evolution and insight development

### Critical Elements for AI

- Exploration approaches for each question

- Evolution patterns and insight development
