## Collaborator Profile

**Purpose:** Document human working patterns, preferences, and expertise to calibrate collaboration.

**Key Sections:**
- Technical Background -- skills, experience, domain knowledge
- Tool Proficiency -- dev tools, knowledge management, analysis tools
- Working Style -- problem-solving approach, learning patterns, cognitive preferences
- Communication Preferences -- interaction style, documentation, feedback patterns

**Usage:**
1. Read at session start to understand human preferences.
2. Adapt communication and interaction style accordingly.
3. Maintain and update as patterns evolve.
4. Use to customize other framework components.

**Critical Elements:** Working style (detailed vs. high-level, structured vs. flexible), communication patterns, learning objectives, tool preferences and constraints.

## Project Foundations

**Purpose:** Establish project parameters, objectives, and quality standards for focused collaboration.

**Key Sections:**
- Purpose and Scope -- objectives, boundaries, success criteria
- Context and Constraints -- environment, limitations, requirements
- Quality Framework -- standards, evaluation criteria, review processes
- Success Metrics -- measurement approaches, milestones, completion criteria

**Usage:**
1. Align all suggestions with stated objectives.
2. Reference constraints when evaluating options.
3. Maintain quality standards throughout work.
4. Track progress against defined metrics.

**Critical Elements:** Primary objective, key constraints, quality standards, scope boundaries (included/excluded).

## Session Template

**Purpose:** Structure individual sessions for consistency, documentation, and progress tracking.

**Key Sections:**
- Session Planning -- objectives, approach, time allocation
- Work Documentation -- activities, decisions, outputs
- Progress Tracking -- accomplishments, open items, next steps
- Reflection and Evolution -- learning, adjustments, improvements

**Usage:**
1. Follow established session structure.
2. Maintain documentation throughout the session.
3. Support progress tracking and planning.
4. Contribute to reflection and improvement.

**Critical Elements:** Session objectives and success criteria, documentation requirements, progress tracking, handoff preparation for continuity.

---

## Living Questions

**Purpose:** Maintain and evolve key questions that drive project development and learning.

**Key Sections:**
- Core Questions -- fundamental project inquiries
- Emerging Questions -- new areas of exploration
- Resolved Insights -- answered questions and learning gained
- Question Evolution -- how inquiries develop over time

**Usage:**
1. Reference questions to guide analysis and suggestions.
2. Identify new questions emerging from work.
3. Support exploration of key questions.
4. Contribute to question evolution and insight development.

**Critical Elements:** Current priority questions, exploration approaches, connections between questions, evolution patterns.

---

## Engagement Guidelines

**Purpose:** Define collaboration patterns, communication approaches, and working relationship norms.

**Key Sections:**
- Communication Patterns -- interaction style, feedback approaches
- Decision Processes -- how choices are made and implemented
- Quality Assurance -- review processes, evaluation methods
- Evolution Mechanisms -- how guidelines adapt and improve

**Usage:**
1. Follow established communication and interaction patterns.
2. Respect decision-making processes and human authority.
3. Support quality assurance per defined approaches.
4. Contribute to guideline evolution when appropriate.

**Critical Elements:** Preferred communication styles, decision-making authority and process, quality evaluation approaches, boundary expectations and role definitions.

## Process Components

**Purpose:** Operational elements supporting framework implementation and maintenance.

**Key Sections:**
- Setup Processes -- initialization, template completion, configuration
- Operational Processes -- session management, documentation, quality control
- Evolution Processes -- review, refinement, adaptation, growth
- Integration Processes -- component connections, information flow

**Usage:**
1. Execute processes according to established patterns.
2. Maintain process consistency and quality.
3. Identify process improvement opportunities.
4. Facilitate integration between process elements.

**Critical Elements:** Process sequence and dependencies, quality checkpoints, documentation requirements, integration and handoff points.

## Navigation Paths

**New Projects:**
Collaborator Profile -> Project Foundations -> Session Template.
Focus: Support human through setup while learning preferences.

**Ongoing Projects:**
Session Context -> Living Questions -> Current Work.
Focus: Restore context and continue established patterns.

**Framework Learning:**
Basics -> Architecture -> Practice.
Focus: Explain concepts and support practical implementation.

**Problem-Solving:**
Troubleshooting -> Relevant Cards -> Adaptation.
Focus: Diagnose issues and support solutions.

## Recovery Procedures

**Lost or confused:**
1. Check Session Context for current state.
2. Review Project Foundations for direction.
3. Reference Collaborator Profile for preferences.

**Quality concerns:**
1. Reference Project Foundations quality standards.
2. Check Engagement Guidelines for process clarity.
3. Document concerns in Session Template.

**Unclear boundaries:**
1. Reference Engagement Guidelines.
2. Ask the human for clarification.
3. Document the pattern for future reference.

**Templates need updates:**
1. Check Evolution Tracking sections.
2. Identify what is not working.
3. Suggest specific improvements and test incrementally.

**LLM behavior:**
- Follow appropriate role boundaries.
- Maintain consistent documentation.
- Support human strategic thinking.
- Contribute to framework evolution.

**Framework health:**
- Clear session continuity.
- Consistent quality standards.
- Effective knowledge development.
- Sustainable collaboration patterns.
